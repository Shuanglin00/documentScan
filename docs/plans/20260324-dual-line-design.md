# 文档扫描与智能检索系统 - 双主线功能设计

> 核心目标：支持两条独立任务线
> - 任务线1：数据入库（文档 / 持续对话）
> - 任务线2：检索问答（针对用户场景）

## 任务线1：数据入库

### 1.1 文档入库流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          文档入库详细流程                                    │
└─────────────────────────────────────────────────────────────────────────────┘

阶段1：文档接收
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：文档文件(PDF/Word/TXT/EPUB)                                           │
│                                                                              │
│ 处理：                                                                        │
│   1. 格式校验（支持的格式？）                                                 │
│   2. 大小校验（<50MB？）                                                     │
│   3. 生成文档ID                                                              │
│   4. 存储原始文件                                                            │
│                                                                              │
│ 输出：DocumentRecord {id, name, type, status=PENDING}                        │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段2：文档解析
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：DocumentRecord                                                         │
│                                                                              │
│ 处理：                                                                        │
│   1. 根据格式选择解析器                                                       │
│   2. 提取纯文本内容                                                          │
│   3. 提取段落结构（段落边界、页码）                                           │
│   4. 提取元数据（标题、作者等）                                               │
│                                                                              │
│ 输出：ParsedDocument {id, content, paragraphs[], metadata}                   │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段3：智能分块
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：ParsedDocument                                                         │
│                                                                              │
│ 处理：                                                                        │
│   1. 按段落切分（优先）                                                       │
│   2. 大段落按句子切分（备选）                                                 │
│   3. 每块最大500字                                                           │
│   4. 保留20%重叠区域（上下文关联）                                            │
│   5. 生成分块索引（0,1,2...）                                                 │
│                                                                              │
│ 输出：Chunk[] {id, documentId, content, overlapPrefix, overlapSuffix,        │
│               startOffset, endOffset, pageNum, chunkIndex}                   │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段4：实体提取
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：Chunk[]                                                                │
│                                                                              │
│ 处理：                                                                        │
│   1. 遍历所有分块提取实体                                                     │
│   2. 识别实体类型（人名/地点/组织/术语/概念）                                  │
│   3. 聚合同名实体                                                            │
│   4. 生成实体档案（描述+别名+提及位置）                                        │
│                                                                              │
│ 输出：Entity[] {id, documentId, name, type, description, aliases,            │
│                mentions[]}                                                   │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段5：向量化与入库
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：Chunk[] + Entity[]                                                     │
│                                                                              │
│ 处理：                                                                        │
│   分块向量化：                                                                │
│     1. 生成稠密向量（语义理解，1024维）                                        │
│     2. 生成稀疏向量（关键词匹配，BM25）                                        │
│     3. 存入Milvus                                                            │
│                                                                              │
│   实体向量化：                                                                │
│     1. 生成稀疏向量（便于精确匹配）                                            │
│     2. 存入Milvus                                                            │
│                                                                              │
│   原文入库：                                                                  │
│     1. 文档元数据 → MongoDB                                                   │
│     2. 分块原文 → MongoDB                                                     │
│     3. 实体信息 → MongoDB                                                     │
│                                                                              │
│ 输出：入库完成状态                                                            │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 持续对话入库流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          对话入库详细流程                                    │
└─────────────────────────────────────────────────────────────────────────────┘

场景：用户与系统的多轮对话需要作为知识入库

流程：
┌─────────────────────────────────────────────────────────────────────────────┐
│                                                                              │

│   ┌─────────────┐                                                           │
│   │ 对话窗口ID  │                                                           │
│   │ 创建时间    │                                                           │
│   │ 参与者      │                                                           │
│   │ 主题/标题   │                                                           │
│   └─────────────┘                                                           │
│          │                                                                   │
│          ▼                                                                   │
│   对话消息流                                                                  │
│   ┌──────────────────────────────────────────────────────────────┐          │
│   │ 消息1：用户提问                                              │          │
│   │ 消息2：AI思考（系统内部）                                     │          │
│   │ 消息3：AI回答                                                │          │
│   │ 消息4：用户追问                                              │          │
│   │ 消息5：AI思考                                                │          │
│   │ 消息6：AI回答                                                │          │
│   └──────────────────────────────────────────────────────────────┘          │
│          │                                                                   │
│          ▼                                                                   │
│   对话分块处理                                                                │
│   ┌──────────────────────────────────────────────────────────────┐          │
│   │ 1. 按消息角色分块（用户/AI）                                  │          │
│   │ 2. 长消息按段落切分                                           │          │
│   │ 3. 标记消息顺序和时间                                         │          │
│   │ 4. 保留对话上下文关联                                         │          │
│   └──────────────────────────────────────────────────────────────┘          │
│          │                                                                   │
│          ▼                                                                   │
│   对话块类型标记                                                              │
│   ┌──────────────────────────────────────────────────────────────┐          │
│   │ DIALOGUE_USER    - 用户提问内容                               │          │
│   │ DIALOGUE_AI_THINKING - AI思考过程                             │          │
│   │ DIALOGUE_AI_ANSWER   - AI最终回答                             │          │
│   └──────────────────────────────────────────────────────────────┘          │
│          │                                                                   │
│          ▼                                                                   │
│   对话实体提取                                                                │
│   ┌──────────────────────────────────────────────────────────────┐          │
│   │ 从对话中提取：                                                │          │
│   │ • 讨论的主题/概念                                             │          │
│   │ • 提到的实体（人名/项目/技术）                                 │          │
│   │ • 形成的共识/结论                                             │          │
│   └──────────────────────────────────────────────────────────────┘          │
│          │                                                                   │
│          ▼                                                                   │
│   对话数据入库                                                                │
│   ┌──────────────────────────────────────────────────────────────┐          │
│   │ MongoDB：                                                     │          │
│   │   • 对话窗口元数据                                            │          │
│   │   • 对话消息原文                                              │          │
│   │   • 提取的实体信息                                            │          │
│   │                                                              │          │
│   │ Milvus：                                                      │          │
│   │   • 对话内容的稠密向量（语义检索）                             │          │
│   │   • 对话内容的稀疏向量（关键词检索）                           │          │
│   └──────────────────────────────────────────────────────────────┘          │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

业务规则：
1. 对话窗口作为虚拟文档，分配唯一ID
2. 对话消息按角色分块，标记类型
3. 保留时间顺序，支持时序检索
4. 对话中的结论性内容作为实体提取
5. 支持对话窗口的合并、拆分
```

## 任务线2：检索问答

### 2.1 检索流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          检索问答详细流程                                    │
└─────────────────────────────────────────────────────────────────────────────┘

阶段1：用户提问接收
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：用户自然语言问题                                                       │
│                                                                              │
│ 示例：                                                                       │
│   "这个API的调用限制是什么？"                                                 │
│   "主角的成长经历有哪些关键节点？"                                             │
│   "之前我们讨论过这个问题的哪方面？"                                           │
│                                                                              │
│ 处理：                                                                        │
│   1. 接收用户问题                                                            │
│   2. 创建Plan（执行计划）                                                    │
│   3. 初始化上下文                                                            │
│                                                                              │
│ 输出：Plan {id, query, status=PENDING, currentDepth=0}                       │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段2：语义分析
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：用户问题                                                               │
│                                                                              │
│ 处理：                                                                        │
│   1. 关键词提取（核心术语）                                                   │
│   2. 查询意图识别（事实查询/定义查询/比较查询）                                 │
│   3. 问题分解（复杂问题拆分为子问题）                                         │
│   4. 实体识别（问题中提到的实体）                                             │
│                                                                              │
│ 输出：AnalysisResult {keywords[], intent, subQueries[], entities[]}          │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段3：多路检索
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：AnalysisResult                                                         │
│                                                                              │
│ 处理：                                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │ 并行执行三路检索：                                                    │  │
│   │                                                                      │  │
│   │ 路1：语义检索（稠密向量）                                              │  │
│   │   - 用户问题向量化                                                    │  │
│   │   - Milvus相似度搜索                                                  │  │
│   │   - 返回Top10                                                         │  │
│   │                                                                      │  │
│   │ 路2：关键词检索（稀疏向量）                                            │  │
│   │   - 提取的关键词向量化                                                │  │
│   │   - BM25检索                                                          │  │
│   │   - 返回Top10                                                         │  │
│   │                                                                      │  │
│   │ 路3：精确检索（可选）                                                  │  │
│   │   - 精确匹配短语                                                      │  │
│   │   - 返回匹配结果                                                      │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                      │                                      │
│                                      ▼                                      │
│   结果融合（加权排序）                                                        │
│   ┌─────────────────────────────────────────────────────────────────────┐  │
│   │ 语义检索结果权重：70%                                                  │  │
│   │ 关键词检索结果权重：30%                                                │  │
│   │                                                                      │  │
│   │ 融合公式：score = semantic_score * 0.7 + keyword_score * 0.3          │  │
│   │                                                                      │  │
│   │ 去重合并 → 重新排序 → 取Top10                                          │  │
│   └─────────────────────────────────────────────────────────────────────┘  │
│                                                                              │
│ 输出：ChunkScore[] {chunkId, content, score, documentId}                     │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段4：答案生成
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：ChunkScore[] + 用户问题                                                │
│                                                                              │
│ 处理：                                                                        │
│   1. 组装Prompt                                                             │
│      - 角色定位（文档助手）                                                   │
│      - 任务目标（基于文档回答问题）                                           │
│      - 上下文记忆（检索到的Top10分块）                                        │
│      - 绝对记忆（提取的实体设定）                                             │
│      - 约束（语言、长度、格式）                                               │
│      - 用户问题                                                              │
│                                                                              │
│   2. 调用LLM生成答案                                                         │
│      - 设置温度参数（0.3-0.7）                                                │
│      - 设置最大长度                                                          │
│      - 超时控制（30秒）                                                       │
│                                                                              │
│   3. 后处理                                                                  │
│      - 提取答案文本                                                          │
│      - 保留引用标记                                                          │
│                                                                              │
│ 输出：Answer {content, references[], confidence}                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
阶段5：偏离度评估与补充检索
┌─────────────────────────────────────────────────────────────────────────────┐
│ 输入：生成的Answer + 用户问题 + 上下文                                        │
│                                                                              │
│ 处理：                                                                        │
│   1. 计算偏离度                                                              │
│      - 答案与问题的语义相似度                                                  │
│      - 检索内容覆盖度                                                         │
│      - 综合偏离度评分                                                         │
│                                                                              │
│   2. 判断是否触发补充检索                                                     │
│      ┌─────────────────────────────────────────────────────────────────┐   │
│      │ 条件判断：                                                        │   │
│      │                                                                  │   │
│      │ IF 偏离度 > 0.7 AND 当前深度 < 3 THEN                             │   │
│      │    - 生成新的关键词                                               │   │
│      │    - 创建新的Step                                                 │   │
│      │    - 执行补充检索                                                 │   │
│      │ ELSE                                                              │   │
│      │    - 返回答案                                                     │   │
│      │ END IF                                                            │   │
│      └─────────────────────────────────────────────────────────────────┘   │
│                                                                              │
│   3. 深度递归控制                                                            │
│      - 每层最多3个Step                                                      │
│      - 每个Step最多3个Task                                                  │
│      - 总深度不超过3层                                                      │
│      - 新关键词必须未搜索过（去重）                                           │
│      - 答案改善幅度<5%时停止                                                 │
│                                                                              │
│ 输出：最终答案 或 新的检索任务                                               │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 两种检索场景

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          场景A：文档检索                                     │
│                    （针对已入库的文档）                                      │
└─────────────────────────────────────────────────────────────────────────────┘

用户场景：
- 查询某个技术规范的合规要求
- 了解小说中某角色的完整设定
- 查找多个文档的关联信息

检索范围：
┌─────────────────────────────────────────────────────────────────────────────┐
│ 可选范围：                                                                   │
│   1. 单文档检索 - 在指定文档内搜索                                            │
│   2. 多文档检索 - 在多个指定文档内搜索                                         │
│   3. 全库检索   - 在所有文档中搜索                                            │
│                                                                              │
│ 过滤条件：                                                                   │
│   - 文档类型（技术文档/小说/对话）                                            │
│   - 文档标签                                                                 │
│   - 上传时间范围                                                             │
└─────────────────────────────────────────────────────────────────────────────┘

示例：
用户："在《API设计规范》中，认证要求有哪些？"

用户："对比《API规范》和《安全规范》的认证要求差异"
系统：在 documentId=["api-spec-v3", "security-spec-v2"] 中检索
```

```
┌─────────────────────────────────────────────────────────────────────────────┐

用户场景：

```

## 3. 双主线交互关系

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          数据入库与检索的关联                                │
└─────────────────────────────────────────────────────────────────────────────┘

                              ┌─────────────┐
                              │   数据入库   │
                              │   任务线    │
                              └──────┬──────┘
                                     │
                                     │ 生成
                                     │
                                     ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                              统一存储层                                      │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐                     │
│  │  MongoDB    │    │   Milvus    │    │   Redis     │                     │
│  │  (原文)     │    │  (向量)     │    │  (上下文)   │                     │
│  │             │    │             │    │             │                     │
│  │ • 文档      │    │ • 稠密向量   │    │ • Plan缓存  │                     │
│  │ • 分块      │    │ • 稀疏向量   │    │ • 会话状态  │                     │
│  │ • 实体      │    │             │    │             │                     │
│  │ • 对话      │    │             │    │             │                     │
│  └─────────────┘    └─────────────┘    └─────────────┘                     │
└─────────────────────────────────────────────────────────────────────────────┘
                                     │
                                     │ 检索
                                     │
                                     ▼
                              ┌─────────────┐
                              │   检索问答   │
                              │   任务线    │
                              └─────────────┘

关键交互点：

1. 入库的数据可被检索
   - 文档分块 → 向量索引 → 语义/关键词检索
   - 对话记录 → 向量索引 → 对话历史检索

2. 检索的结果反馈入库
   - 问答过程中发现的新实体 → 补充到实体库
   - 有价值的问答对 → 可作为知识入库

3. 版本管理影响检索
   - 文档更新版本 → 新版本向量入库
   - 检索时指定版本范围

4. 对话的循环增强
   - 用户对话 → 入库
   - 后续检索 → 可检索到历史对话
   - 形成知识沉淀
```

## 4. 核心业务流程总结

### 数据入库线（异步处理）

```
文档/对话 → 解析 → 分块 → 实体提取 → 向量化 → 入库(MongoDB+Milvus)
  │           │       │         │         │            │
  │           │       │         │         │            │
  ▼           ▼       ▼         ▼         ▼            ▼
接收        格式     段落      命名      稠密+稀疏    原文+向量
校验        识别     切分      实体      向量生成     持久化
```

### 检索问答线（实时处理）

```
用户提问 → 语义分析 → 多路检索 → 组装Prompt → LLM生成 → 偏离度评估 → 返回答案
   │           │          │            │           │           │
   │           │          │            │           │           │
   ▼           ▼          ▼            ▼           ▼           ▼
创建Plan   关键词+    语义70%+     四级记忆    流式输出    偏离度>0.7
           意图       关键词30%    结构        答案生成    触发补充
```
